# 03 · 数据库设计（v1.1 开发期修订）

> **可执行文件在 `db/` 目录**：`schema.sql`（本文档 DDL + settings 初始值）→
> `seed_regions.sql` → `seed_categories.sql`；旧库升级用 `db/migrate_*.sql`。

> 把所有数据对象落成可直接建库的字段、类型、索引与建表语句。MySQL 8.x，表前缀 `zhaopin_`。

---

## 1. 通用约定

| 项 | 约定 | 理由 |
|----|------|------|
| 存储引擎 | InnoDB | 事务、外键、行锁 |
| 字符集 | `utf8mb4` | 中文+emoji |
| 排序规则 | `utf8mb4_0900_ai_ci` | MySQL 8 默认 |
| 时间 | `DATETIME`，统一存 **UTC** | 应用层转马德里时区 |
| IP | `VARBINARY(16)` | `INET6_ATON/NTOA`，兼容 IPv4/IPv6 |
| 主键 | 自增整数 | 内部用，**永不出现在 URL/页面/API** |
| 对外标识 | 随机 `public_code`/`code` | 不可枚举，防 IDOR |
| 布尔 | `TINYINT UNSIGNED`(0/1) | — |
| 机密 | **不入库** | 放 `app/config/config.php` |

命名：表 `zhaopin_xxx`；索引 `idx_*`、唯一键 `uk_*`。

## 3. 字段定义

### 3.1 `zhaopin_posts`（信息表，核心）
| 字段 | 类型 | 空 | 默认 | 说明 |
|------|------|----|------|------|
| id | BIGINT UNSIGNED | 否 | AI | 内部主键 |
| public_code | CHAR(10) | 否 | | 对外随机标识（详情页 URL，不可枚举） |
| type | TINYINT UNSIGNED | 否 | | 1=招聘 2=求职 |
| content | VARCHAR(1000) | 否 | | 信息内容（限长防灌水） |
| content_hash | CHAR(64) | 否 | | 规整内容 SHA-256，用于同内容判重 |
| contact_name | VARCHAR(50) | 否 | | 联系人/姓名 |
| phone | VARCHAR(30) | 否 | | 电话（展示用，原样） |
| phone_norm | VARCHAR(20) | 否 | | 规整电话（去符号），判重用 |
| wechat | VARCHAR(60) | 是 | NULL | 微信号（选填） |
| region_id | INT UNSIGNED | 否 | | → regions |
| category_id | INT UNSIGNED | 否 | | → categories |
| poster_type | TINYINT UNSIGNED | 否 | | 1=游客 2=注册用户 |
| user_id | BIGINT UNSIGNED | 是 | NULL | 注册用户→users；游客 NULL |
| is_top | TINYINT UNSIGNED | 否 | 0 | 是否置顶 |
| top_expire_at | DATETIME | 是 | NULL | 置顶到期 |
| invalid_count | INT UNSIGNED | 否 | 0 | 失效标记累计 |
| phone_views | INT UNSIGNED | 否 | 0 | 电话被查看累计次数 |
| wechat_views | INT UNSIGNED | 否 | 0 | 微信被查看累计次数 |
| report_count | INT UNSIGNED | 否 | 0 | 举报累计（冗余） |
| suspicious | TINYINT UNSIGNED | 否 | 0 | 1=可疑（后台高亮） |
| status | TINYINT UNSIGNED | 否 | 1 | 1=在线 0=到期下架 2=主动下架 3=管理员删除 |
| post_ip | VARBINARY(16) | 是 | NULL | 发布 IP |
| created_at | DATETIME | 否 | | 发布时间 |
| bumped_at | DATETIME | 否 | | 顶帖时间（排序键之一） |
| updated_at | DATETIME | 是 | NULL | 最后修改 |

**索引**：PRIMARY(id)；uk_public_code(public_code)；idx_browse(type,status,is_top,bumped_at)；idx_browse_cat(type,status,category_id,bumped_at)；idx_browse_region(type,status,region_id,bumped_at)；idx_dedup(phone_norm,content_hash)；idx_user(user_id,status)；idx_top(is_top,top_expire_at)。
> 排序 `is_top DESC,bumped_at DESC` 两列同向，升序复合索引可反向扫描满足。类别+地区同时筛选的复合索引按真实流量用 EXPLAIN 再调。

### 3.5 `zhaopin_users`
| 字段 | 类型 | 说明 |

| display_name | VARCHAR(60) NULL | 显示名 |

### 3.6 `zhaopin_admins`（白名单，v1.1 增加分级）
| 字段 | 类型 | 说明 |

**索引**：PRIMARY(id)；uk_email(email)。
> 登录校验：Google 通过且命中白名单才授权；全站零密码；管理员账号须开 2FA。

**索引**：PRIMARY(id)；idx_post(post_id)；idx_status(status)；idx_dedup(post_id,reporter_ip,created_at)

### 3.8 `zhaopin_coupons`（置顶券）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT UNSIGNED AI | 内部主键 |
| code | VARCHAR(20) | 兑换码（随机不可枚举） |
| top_days | SMALLINT UNSIGNED | 置顶时长(天) |
| valid_until | DATETIME | 券有效期(未兑换则作废) |
| status | TINYINT UNSIGNED | 0未兑换 1已兑换未用 2已使用 3已过期 4已作废 |
| user_id | BIGINT UNSIGNED NULL | 兑换归属 |
| post_id | BIGINT UNSIGNED NULL | 使用所在帖 |
| created_by | INT UNSIGNED NULL | 生成管理员→admins |
| created_at / redeemed_at / used_at | DATETIME | 各阶段时间 |

**索引**：PRIMARY(id)；uk_code(code)；idx_user(user_id,status)；idx_status(status)。
> 生命周期：生成(0)→兑换(1,写user_id)→用于某帖(2,写post_id)。本期不防转赠。

### 3.9 `zhaopin_invalid_marks`

### 3.10 `zhaopin_keywords`
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT UNSIGNED AI | 主键 |
| word | VARCHAR(50) | 敏感词 |

**索引**：PRIMARY(id)；uk_word(word)；idx_type_status(type,status)

### 3.11 `zhaopin_settings`（键值）
| 字段 | 类型 | 说明 |
|------|------|------|
| skey | VARCHAR(64) | 参数键（主键） |
| svalue | TEXT NULL | 值（标量或 JSON） |
| updated_at | DATETIME NULL | |

**初始键（默认值，后台"参数配置"带中文说明可改）**：post_expire_days=30｜backfill_n_recruit=30｜backfill_n_seek=30｜daily_post_limit=1｜bump_window_days=7｜top_slot_limit=5｜report_dedup_window_min=60｜report_email_merge_min=60｜report_recipients=[](JSON)｜invalid_threshold=5｜contact_reveal_per_hour=30｜post_per_hour_per_ip=5｜field_name_map=(JSON)｜**normal_admin_can_coupon=0**（v1.1：普通管理员发券开关，超管在"管理员"页切换）。
> **机密（Brevo Key、OAuth secret）不放此表**，放 `app/config`。

### 3.12 `zhaopin_contact_log`（取号日志）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT UNSIGNED AI | 主键 |
| post_id | BIGINT UNSIGNED | →posts |
| contact_type | TINYINT UNSIGNED | 1=电话 2=微信（分开计数用） |
| viewer_ip | VARBINARY(16) | 取号 IP |
| created_at | DATETIME | |

**索引**：PRIMARY(id)；idx_ip_time(viewer_ip,created_at)；idx_post_type(post_id,contact_type)。可定期清理(只留 30 天)。
> 计数口径：`posts.phone_views/wechat_views` 累计供前台快速展示；`contact_log` 明细供后台算去重"N 人查看"与电话/微信偏好分析。前台取号时前端按类型（phone/wechat）分别调接口。

## 4. 关系概览
posts.user_id→users.id（游客 NULL）｜posts.region_id→regions.id｜posts.category_id→categories.id｜reports/invalid_marks/contact_log/coupons.post_id→posts.id｜coupons.user_id→users.id、coupons.created_by→admins.id｜regions.parent_id→regions.id(自引用,0顶级)。
> 外键约束是否物理建立可选：核心关系建 FK 更严谨；日志类(contact_log)不建以免高频写入受约束。列入待核对。

## 5. 建表 DDL（可直接执行）

```sql
SET NAMES utf8mb4;

CREATE TABLE zhaopin_regions (
  id INT UNSIGNED NOT NULL AUTO_INCREMENT,
  parent_id INT UNSIGNED NOT NULL DEFAULT 0,
  name VARCHAR(50) NOT NULL,
  level TINYINT UNSIGNED NOT NULL,
  sort INT NOT NULL DEFAULT 0,
  status TINYINT UNSIGNED NOT NULL DEFAULT 1,
  PRIMARY KEY (id),
  KEY idx_parent (parent_id, status, sort)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci;

CREATE TABLE zhaopin_categories (
  id INT UNSIGNED NOT NULL AUTO_INCREMENT,
  name VARCHAR(30) NOT NULL,
  sort INT NOT NULL DEFAULT 0,
  status TINYINT UNSIGNED NOT NULL DEFAULT 1,
  created_at DATETIME NOT NULL,
  PRIMARY KEY (id),
  UNIQUE KEY uk_name (name),
  KEY idx_status_sort (status, sort)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci;

CREATE TABLE zhaopin_categories_pending (
  id INT UNSIGNED NOT NULL AUTO_INCREMENT,
  name VARCHAR(30) NOT NULL,
  submit_ip VARBINARY(16) DEFAULT NULL,
  user_id BIGINT UNSIGNED DEFAULT NULL,
  status TINYINT UNSIGNED NOT NULL DEFAULT 0,
  submitted_at DATETIME NOT NULL,
  reviewed_at DATETIME DEFAULT NULL,
  PRIMARY KEY (id),
  KEY idx_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci;

CREATE TABLE zhaopin_users (
  id BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
  public_code CHAR(8) NOT NULL,
  google_sub VARCHAR(64) NOT NULL,
  display_name VARCHAR(60) DEFAULT NULL,
  status TINYINT UNSIGNED NOT NULL DEFAULT 1,
  created_at DATETIME NOT NULL,
  last_login_at DATETIME DEFAULT NULL,
  PRIMARY KEY (id),
  UNIQUE KEY uk_google_sub (google_sub),
  UNIQUE KEY uk_public_code (public_code)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci;

CREATE TABLE zhaopin_admins (
  id INT UNSIGNED NOT NULL AUTO_INCREMENT,
  email VARCHAR(255) NOT NULL,
  role TINYINT UNSIGNED NOT NULL DEFAULT 1, -- 1=普通管理员 2=超级管理员
  google_sub VARCHAR(64) DEFAULT NULL,
  display_name VARCHAR(60) DEFAULT NULL,
  status TINYINT UNSIGNED NOT NULL DEFAULT 1,
  created_at DATETIME NOT NULL,
  PRIMARY KEY (id),
  UNIQUE KEY uk_email (email)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci;

CREATE TABLE zhaopin_posts (
  id BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
  public_code CHAR(10) NOT NULL,
  type TINYINT UNSIGNED NOT NULL,
  content VARCHAR(1000) NOT NULL,
  content_hash CHAR(64) NOT NULL,
  contact_name VARCHAR(50) NOT NULL,
  phone VARCHAR(30) NOT NULL,
  phone_norm VARCHAR(20) NOT NULL,
  wechat VARCHAR(60) DEFAULT NULL,
  region_id INT UNSIGNED NOT NULL,
  category_id INT UNSIGNED NOT NULL,
  poster_type TINYINT UNSIGNED NOT NULL,
  user_id BIGINT UNSIGNED DEFAULT NULL,
  is_top TINYINT UNSIGNED NOT NULL DEFAULT 0,
  top_expire_at DATETIME DEFAULT NULL,
  invalid_count INT UNSIGNED NOT NULL DEFAULT 0,
  phone_views INT UNSIGNED NOT NULL DEFAULT 0,
  wechat_views INT UNSIGNED NOT NULL DEFAULT 0,
  report_count INT UNSIGNED NOT NULL DEFAULT 0,
  suspicious TINYINT UNSIGNED NOT NULL DEFAULT 0,
  status TINYINT UNSIGNED NOT NULL DEFAULT 1,
  post_ip VARBINARY(16) DEFAULT NULL,
  created_at DATETIME NOT NULL,
  bumped_at DATETIME NOT NULL,
  updated_at DATETIME DEFAULT NULL,
  PRIMARY KEY (id),
  UNIQUE KEY uk_public_code (public_code),
  KEY idx_browse (type, status, is_top, bumped_at),
  KEY idx_browse_cat (type, status, category_id, bumped_at),
  KEY idx_browse_region (type, status, region_id, bumped_at),
  KEY idx_dedup (phone_norm, content_hash),
  KEY idx_user (user_id, status),
  KEY idx_top (is_top, top_expire_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci;

CREATE TABLE zhaopin_reports (
  id BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
  post_id BIGINT UNSIGNED NOT NULL,
  reason VARCHAR(200) DEFAULT NULL,
  reporter_ip VARBINARY(16) DEFAULT NULL,
  status TINYINT UNSIGNED NOT NULL DEFAULT 0,
  created_at DATETIME NOT NULL,
  handled_at DATETIME DEFAULT NULL,
  PRIMARY KEY (id),
  KEY idx_post (post_id),
  KEY idx_status (status),
  KEY idx_dedup (post_id, reporter_ip, created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci;

CREATE TABLE zhaopin_coupons (
  id BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
  code VARCHAR(20) NOT NULL,
  top_days SMALLINT UNSIGNED NOT NULL,
  valid_until DATETIME NOT NULL,
  status TINYINT UNSIGNED NOT NULL DEFAULT 0,
  user_id BIGINT UNSIGNED DEFAULT NULL,
  post_id BIGINT UNSIGNED DEFAULT NULL,
  created_by INT UNSIGNED DEFAULT NULL,
  created_at DATETIME NOT NULL,
  redeemed_at DATETIME DEFAULT NULL,
  used_at DATETIME DEFAULT NULL,
  PRIMARY KEY (id),
  UNIQUE KEY uk_code (code),
  KEY idx_user (user_id, status),
  KEY idx_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci;

CREATE TABLE zhaopin_invalid_marks (
  id BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
  post_id BIGINT UNSIGNED NOT NULL,
  marker_ip VARBINARY(16) NOT NULL,
  created_at DATETIME NOT NULL,
  PRIMARY KEY (id),
  UNIQUE KEY uk_post_ip (post_id, marker_ip),
  KEY idx_post (post_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci;

CREATE TABLE zhaopin_keywords (
  id INT UNSIGNED NOT NULL AUTO_INCREMENT,
  word VARCHAR(50) NOT NULL,
  type TINYINT UNSIGNED NOT NULL DEFAULT 1,
  status TINYINT UNSIGNED NOT NULL DEFAULT 1,
  created_at DATETIME NOT NULL,
  PRIMARY KEY (id),
  UNIQUE KEY uk_word (word),
  KEY idx_type_status (type, status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci;

CREATE TABLE zhaopin_settings (
  skey VARCHAR(64) NOT NULL,
  svalue TEXT,
  updated_at DATETIME DEFAULT NULL,
  PRIMARY KEY (skey)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci;

CREATE TABLE zhaopin_contact_log (
  id BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
  post_id BIGINT UNSIGNED NOT NULL,
  contact_type TINYINT UNSIGNED NOT NULL,
  viewer_ip VARBINARY(16) NOT NULL,
  created_at DATETIME NOT NULL,
  PRIMARY KEY (id),
  KEY idx_ip_time (viewer_ip, created_at),
  KEY idx_post_type (post_id, contact_type)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci;
```
