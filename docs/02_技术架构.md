## 1. 技术栈总览

| 层 | 选型 | 说明 |
|----|------|------|
| 语言 | 原生 PHP 8.x（无框架） | 最大化通用性与服务器兼容性 |
| 架构模式 | 注册表（Registry）驱动 + 多入口直链 | 见 §3 |
| 数据库 | MySQL 8.x | 表前缀 `zhaopin_` |
| 前端 | Bootstrap 5 + 原生 JS | 通用技术，无构建链 |
| 渲染 | **服务端渲染为主 + API 为辅** | 页面 PHP 直出 HTML（利于 SEO）；动作走 API |
| 登录 | Google OAuth | 零成本；普通用户与管理员（白名单）均用 |
| 邮件 | Brevo 事务邮件 | 免费档 + 域名发件 |

**总原则**：通用、可移植、无重型依赖，标准 LAMP/LEMP 即可运行。

## 2. 目录结构

```
虚拟主机根目录/
├── zp_html/                  ← 网站根目录(document root)，URL 可访问
│   ├── .htaccess             无扩展名 URL 重写 + index/.php 301 归一（Apache）
│   ├── index.php             首页(信息墙)，URL 即 /
│   ├── list.php  detail.php  publish.php  privacy.php     页面薄壳
│   ├── report.php  mark_invalid.php  get_contact.php      动作 API 薄壳
│   ├── user/                 用户后台（login/logout/my_posts/edit/redeem/use_coupon.php）
│   ├── c/cp/                 管理后台（约定入口 /c/cp/）：index/login/logout/posts/
│   │                         post_edit/categories/reports/invalid/coupons/admins/settings.php
│   └── assets/               静态资源：css/（site+页面级）、js/（页面级）、fonts/（本地字体）、img/
│
├── app/                      ← URL 不可访问(在 document root 之外，另有 .htaccess 双保险)
│   ├── registry.php          注册表：功能名 → 处理器文件 映射(兼白名单)
│   ├── bootstrap.php         公共启动：配置/公共库/注册表 + 全局异常兜底(对外零内部信息)
│   ├── config/config.php     机密：DB 密码、OAuth secret、Brevo Key（不入库、不进版本库）
│   ├── content/              静态内容源（privacy.md 隐私政策）
│   ├── handlers/             各功能真正业务逻辑（含 admin/、user/ 子目录）
│   ├── lib/                  公共类库：db/view/response/util/settings/session/auth/posts/mail
│   └── views/                服务端渲染模板（layout/、admin/、user/、页面视图与 _card 局部）
│
├── db/                       建库文件：schema.sql、seed_regions.sql、seed_categories.sql、
│                             migrate_*.sql（旧库升级）
└── tools/dev_router.php      本地开发路由器（php -S 模拟生产重写规则）
```

**安全要点**：机密与业务逻辑全在 `app/`（document root 之外，URL 不可达）；`zp_html/` 内 `.php` 为薄壳入口。

### 2.1 URL 规范（v1.1）
- **无扩展名 URL**：对外一律不带 `.php`（`/publish`、`/detail?id=随机码`、`/c/cp/posts`）。
- **归一化 301**：显式访问 `/xxx.php` → `/xxx`；任意目录下的 `index(.php)` → 目录本身
  （`/index`→`/`，`/c/cp/index`→`/c/cp/`），查询串保留——避免同页多 URL 伤 SEO，也不暴露技术栈。
- 实现：Apache 用 `zp_html/.htaccess`（仅当目标 .php 存在才内部映射，不扩大攻击面）；
  Nginx 配置见 README；本地 `php -S` 用 `tools/dev_router.php` 模拟。

## 3. 注册表（Registry）与请求生命周期

### 3.1 设计意图
不用单一前控制器（拒绝 `index.php?api=xxx`），改用**多个干净直链入口**（`publish.php`、`report.php` …），每个是真实存在的薄壳文件，URL 直观。薄壳不写业务逻辑，经 `registry.php` 找到 `app/handlers/` 内真正处理器执行。

### 3.2 注册表登记内容
登记 **"功能名 → app 处理器文件路径"** 映射；同时兼作**白名单**——仅登记在册的功能名可被调起，杜绝"猜文件名"探测。

### 3.3 单请求生命周期
1. 命中 `zp_html/xxx.php`（薄壳）；2. `require` 上层 `app/bootstrap.php`（载配置/连库/注册表/公共库）；3. 薄壳声明功能名；4. 经注册表定位处理器并执行；5. 处理器跑业务（校验、防护、读写库）；6. 返回：页面类用 `views/` 服务端渲染 HTML，动作/取号类返回 JSON 给前端原生 JS。

## 4. 前后端策略

- **服务端渲染为主**：列表/详情等靠搜索引擎引流的页面由 PHP 直出完整 HTML，保证 SEO；模板在 `app/views/`。
- **API 为辅**：以下动作由原生 JS 经 `fetch` 调对应 `.php` 返回 JSON——点击取号、发布提交、举报、标记失效、顶帖/编辑/删除、兑券/用券、管理员各操作。
- **前端原则**：原生 JS，无构建步骤、无框架、无打包工具；直接引用 `assets/` 静态文件。
- **设计体系（v1.1，按定稿设计模板）**：纸色底 `#FBF7F1` + 珊瑚红(招聘)/玉绿(求职)双主题
  色、卡片式信息墙；公共令牌/组件在 `site.css`，页面级样式脚本按页拆分（home/detail/
  publish/user/admin/privacy），由布局层按页面标识自动加载。
- **字体本地托管（v1.1）**：Bricolage Grotesque + Noto Sans SC 的 woff2 分片存
  `assets/fonts/`，`fonts.css` 保留 unicode-range 子集声明按需加载；不连 Google
  Fonts（GDPR 合规 + 消除第三方请求）。

## 5. 接口清单（v1.1，均为无扩展名直链）

> 页面类服务端渲染 HTML；动作类返回 JSON 给前端原生 JS。

| 入口 | 方法 | 作用 | 防护/权限要点 |
|------|------|------|----------|
| `/` `/list` | GET | 信息墙（?type=job/seek、?region=、?cat=、?q=） | 到期下架/保底N惰性维护 |
| `/detail?id=码` | GET | 信息详情 | 无联系方式明文 |
| `/publish` | GET/POST | 发布页+提交(JSON) | 算术验证码、蜜罐、同IP限频、每日限1条、7天顶帖、正文夹带检测、字段名混淆 |
| `/get_contact` | POST | 取号（code+ctype=phone/wechat） | 同IP每小时上限、同IP同帖同类型去重计数、contact_log |
| `/report` | POST | 举报 | 同IP同帖窗口去重、Brevo 合并降频邮件 |
| `/mark_invalid` | POST | 标记已失效 | 同IP同帖唯一键去重 |
| `/privacy` | GET | 隐私政策（app/content/privacy.md 渲染） | — |
| `/user/login` `/user/logout` | GET | Google 登录/回调、退出 | OAuth state 校验 |
| `/user/my_posts` | GET/POST | 我的发布 + 顶帖(24h限一次)/下架/删除 | 登录态、仅本人、CSRF |

| `/c/cp/settings` | GET/POST | 参数配置（带中文说明） | 仅超管 |

## 6. 防护实现（对应需求 §7：尽力而为，非安全保证）

- **点击取号（防爬核心 + 热度计数）**：列表/详情 HTML 不含联系方式明文；点击时 JS 才请求 `get_contact.php`（带 `type=phone` 或 `type=wechat`），后端校验后返回。因取号是独立请求，可在此加**频率限制**（同 IP 单位时间上限）+ 记 `contact_log`（含类型），异常高频降速/暂拒。同时**按类型对 `posts.phone_views/wechat_views` 累计 +1**（同 IP 同帖同类型短时间内去重、不重复计数），用于展示热度。这是"点击才显示"相较"CSS 隐藏"唯一有防爬意义之处。
- **防机器人**：验证码 + 蜜罐（隐藏框被填即判机器人）+ 频率限制。
- **字段名混淆与轮换**：关键字段用无意义 `name`，当前映射存 `settings`，渲染时读取、提交时还原；轮换只改配置不动代码。
- **可疑标记**：按需求 §7.1 六规则在发布处理器打标；可疑信息照常显示，仅后台高亮。
- **举报邮件提醒**：收件邮箱存 `settings`；同 IP 同帖去重；同帖发信节流（合并窗内只累计不发，过窗再发一封含累计数）。**发信用 Brevo**（API 或 SMTP 中继），域名发件 `no-reply@zhaopin.es`，有发送回执。**不用 163/Yahoo 个人邮箱**（风控严、易封、易静默失败）。
- **置顶上限准入**：用券时查在场置顶数（`is_top=1 且 top_expire_at>now`），<上限才放行，绝不取消已在场者。
- **正文夹带检测**：发布处理器对内容做电话/微信正则检测，命中提示/屏蔽。
- **失效标记**：`mark_invalid.php` 按"同 IP+同帖"去重写入，累计达阈值降权或提醒。

## 7. SEO 实现

> 策略：**主攻需求词**（地区×职位，如"马德里 厨师 招聘""Usera 美甲 招人"），**不硬抢"欧浪"等品牌词**（抢不到且有蹭商标风险）。

- 服务端渲染保证爬虫可读；每条信息独立 URL（对外用不可枚举随机码）+ 语义化标题。
- 招聘详情页嵌入 schema.org **JobPosting** 结构化数据（求职帖不适用，另定方案）。
- 自动生成 `sitemap.xml`；规范 meta；移动友好；加载速度。
- 维护一份地区×职位目标关键词清单，随上线地区扩展。

## 8. 部署与兼容

- 标准 LAMP/LEMP，PHP 8.x + MySQL 8.x；上传即运行（前端无构建）。
- **document root 指向 `zp_html/`**，确认 `app/` 在其外不可访问（app/.htaccess 双保险）。
- **无扩展名 URL 重写**：Apache 由 `zp_html/.htaccess` 自动生效；Nginx 按 README 配置。
- **管理后台**：`/c/cp/`；OAuth 回调登记 `<base_url>/user/login` 与 `<base_url>/c/cp/login`。
- **本地调试**：`php -S 127.0.0.1:8000 -t zp_html tools/dev_router.php`；
  config 的 `dev.fake_admin_email` / `dev.fake_user_name` 可免 OAuth 直登，**生产必须留空**。
- **HTTPS 必须**（OAuth 与取号接口）。
- 机密放 `app/config/config.php`，不入库不进版本库。
- **数据库定期自动备份**。
- **Brevo 配域名 SPF/DKIM**，否则邮件进垃圾箱。
